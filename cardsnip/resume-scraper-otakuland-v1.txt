Comportement du scraper
-----------------------

Le scraper extrait :

- titre produit ;
- prix ;
- stock ;
- source_url ;
- checked_at ;
- target_price.

Il retourne :

ProductCheck

Selecteurs utilises :

Titre :

main.ProductPage h1[itemprop="name"]

Prix :

main.ProductPage .Price-value[itemprop="price"][content]

Stock :

main.ProductPage .ProductStock-info

Stock rupture :

- .ProductStock-info.out_of_stock
- data-sold-out="true"

Stock positif :

- .ProductStock-info.in_stock
- texte "En stock"


Options obligatoires
--------------------

Si une page contient :

isOptionRequired: true

Le scraper refuse proprement avec :

Page Otakuland non supportee en V1 : options obligatoires detectees.

Raison :

Les variantes peuvent avoir des prix et stocks differents.
La V1 ne doit pas deviner.

Bugs corriges pendant validation
--------------------------------

1. /observations/latest

Probleme :

La requete pouvait renvoyer plusieurs observations pour un meme tracked_product si plusieurs checks partageaient le meme timestamp.

Correction :

Selection deterministe via max(id) par tracked_product_id.

2. test-end-to-end-local.ps1

Probleme :

PowerShell pouvait mal manipuler les tableaux JSON retournes par /observations/latest.

Correction :

Ajout d'une fonction Read-JsonArray pour aplatir proprement les reponses tableau.


Limites restantes
-----------------

- pas de support des options/variantes ;
- pas de crawl catalogue ;
- pas de recuperation automatique d'images ;
- pas de recherche interne ;
- pas de pagination boutique ;
- pas de scheduler ;
- pas d'autorisation commerciale confirmee pour usage SaaS public.


Conclusion
----------

Otakuland-Manga Passion peut etre considere functional pour le perimetre V1 limite.

Le statut ne signifie pas :

- scraper universel ;
- support des variantes ;
- support catalogue ;
- usage SaaS public autorise.

Il signifie :

CardSnip sait suivre une URL produit simple Otakuland-Manga Passion, extraire prix/stock, ecrire une observation SQLite, generer les alertes attendues et exposer le resultat via l'API/dashboard.
