Resume - Preparation registry scrapers CardSnip

Objectif de cette passe
-----------------------

Preparer CardSnip a accueillir plusieurs scrapers plus tard, sans coder de vraie boutique maintenant.

Demandes traitees :

1. Clarifier l'interface BaseScraper.
2. S'assurer que FakeShopScraper suit exactement cette interface.
3. Ajouter un champ scraper_key cote shops.
4. Preparer un registry simple des scrapers.
5. Documenter comment ajouter une nouvelle boutique.
6. Ne pas coder de vraie boutique maintenant.


Ce qui a ete modifie
--------------------

1. Interface BaseScraper clarifiee

Fichier modifie :

scraper/scrapers/base.py

Changements :

- ajout du champ de classe obligatoire scraper_key ;
- conservation du constructeur commun :
  - source_url
  - target_price
- clarification du contrat scrape() :
  - chaque scraper doit lire une source ;
  - chaque scraper doit retourner un ProductCheck normalise.

Role :

BaseScraper devient le contrat commun que tous les futurs scrapers devront respecter.


2. FakeShopScraper aligne sur l'interface

Fichier modifie :

scraper/scrapers/fake_shop.py

Changements :

- ajout de :

scraper_key = "fake_shop"

Role :

FakeShopScraper est maintenant identifie par une cle stable et peut etre charge via le registry.

Important :

FakeShopScraper reste le seul scraper fonctionnel actuellement.
Aucune vraie boutique n'a ete ajoutee.


3. Registry simple des scrapers

Fichier ajoute :

scraper/scrapers/registry.py

Contenu principal :

SCRAPER_REGISTRY = {
    "fake_shop": FakeShopScraper,
}

Fonction ajoutee :

create_scraper(scraper_key, source_url, target_price)

Role :

- selectionner la bonne classe de scraper selon shops.scraper_key ;
- instancier le scraper ;
- retourner une erreur claire si la cle est inconnue.

Exemple d'erreur possible :

Scraper inconnu: not_configured. Scrapers disponibles: fake_shop


4. Ajout scraper_key cote SQLite shops

Fichier modifie :

scraper/storage/schema.sql

Changement :

La table shops contient maintenant :

scraper_key text not null default 'not_configured'

Role :

Relier chaque boutique a un scraper disponible, sans coder la logique boutique directement dans tracked_check_service.


5. Migration legere pour DB existante

Utilite de cette passe
----------------------

Avant cette passe :

Le scraper etait encore implicitement lie a FakeShopScraper.

Apres cette passe :

Le systeme sait :

1. lire un tracked_product ;
2. regarder sa boutique ;
3. recuperer shops.scraper_key ;
4. trouver le scraper correspondant dans le registry ;
5. lancer le scraper sans connaitre sa classe en dur.

C'est une base plus propre pour brancher un premier vrai scraper autorise plus tard, sans casser le flux MVP local.


Prochaine etape recommandee
---------------------------

Ne pas ajouter de vraie boutique tout de suite.

Prochaine etape logique :

1. Continuer a stabiliser le MVP local.
2. Ajouter quelques tests Python simples sur le registry.
3. Verifier le flux Catalogue -> suivi -> scraper avec plusieurs suivis fake shop.
4. Garder not_configured pour toutes les boutiques non implementees.
5. Quand une source autorisee sera choisie, creer un nouveau scraper qui herite de BaseScraper et l'enregistrer dans registry.py.
